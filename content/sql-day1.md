# SQL День 1: База с нуля для собеса QA

## Цель дня

- Уверенно писать простые SQL-запросы без пауз и паники.
- Понимать порядок выполнения запроса и объяснять свою логику интервьюеру.
- Закрыть блок: `SELECT`, `WHERE`, `ORDER BY`, `LIMIT`, `DISTINCT`, `COUNT`, `NULL`.

## План дня (6-8 часов SQL-фокуса)

1. `1.5 ч` — синтаксис и логический порядок SQL.
2. `2 ч` — базовые конструкции с примерами.
3. `2 ч` — задачи в тренажёре.
4. `1-2.5 ч` — повтор + устный лайвкодинг.

## 1) Логический порядок выполнения SELECT

Порядок, который полезно помнить:

1. `FROM`
2. `WHERE`
3. `GROUP BY`
4. `HAVING`
5. `SELECT`
6. `ORDER BY`
7. `LIMIT`

Почему это важно:

- помогает понимать, почему нельзя фильтровать агрегат в WHERE;
- помогает отлаживать запрос по этапам.

## 2) Базовые конструкции

### 2.1 SELECT

```sql
SELECT product_name, price
FROM products;
```

### 2.2 WHERE

```sql
SELECT product_name, price
FROM products
WHERE price > 10000;
```

### 2.3 ORDER BY

```sql
SELECT product_name, price
FROM products
ORDER BY price DESC;
```

### 2.4 LIMIT

```sql
SELECT product_name, price
FROM products
ORDER BY price DESC
LIMIT 3;
```

### 2.5 DISTINCT

```sql
SELECT DISTINCT category
FROM products;
```

### 2.6 COUNT

```sql
SELECT COUNT(*) AS total_rows
FROM products;
```

### 2.7 IS NULL / IS NOT NULL

```sql
SELECT *
FROM products
WHERE category IS NULL;
```

## 3) Типовые собесные задачи уровня День 1

1. Вывести товары дороже N.
2. Показать топ-3 по цене.
3. Найти уникальные категории.
4. Посчитать количество строк.
5. Отфильтровать записи с пустым полем.

## 4) Частые ошибки новичка

- `= NULL` вместо `IS NULL`.
- `ORDER BY` без явного направления (ASC/DESC) в задачах с топами.
- Фильтрация не того поля.
- Пропуск alias для читаемости результата.

## 5) Как решать задачу на лайвкодинге

Алгоритм:

1. Проговорить, что нужно получить на выходе.
2. Выбрать таблицу и нужные колонки.
3. Добавить фильтр (если есть).
4. Добавить сортировку.
5. Добавить лимит.
6. Запустить и проверить, что результат выглядит логично.

## 6) Мини-чеклист проверки запроса

- Выбрал правильную таблицу?
- Выбрал только нужные колонки?
- Фильтр соответствует условию задачи?
- Сортировка в верном направлении?
- LIMIT стоит там, где нужен?

## 7) Устный прогон

Проговори 3 решения вслух:

1. «Покажи два самых дорогих товара».
2. «Покажи количество товаров в таблице».
3. «Покажи все уникальные категории, где category не пустая».

## 8) Финал дня

Ты готов к базовому SQL блоку, если:

- пишешь простые запросы без подсказки;
- объясняешь каждую строку запроса;
- находишь и исправляешь типовые ошибки за 1-2 минуты.
