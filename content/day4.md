# 🎯 ДЕНЬ 4: Клиент-сервер, HTTP, API

## База

- Клиент: UI / mobile app / web app
- Сервер: бизнес-логика + доступ к данным
- API: контракт обмена данными между системами

## Что важно

- HTTP-методы: GET / POST / PUT / PATCH / DELETE
- Коды ответов: 2xx / 4xx / 5xx
- Headers, body, auth token
- Идемпотентность
- Валидация схемы ответа

## Что могут дать на логику

"Есть endpoint создания заказа. Какие проверки?"

Проверяем:
- успешный сценарий,
- невалидные данные,
- неавторизованный доступ,
- права пользователя,
- дубликаты / повторный запрос,
- корректность записи в БД,
- консистентность суммы/статуса.

## API-чеклист QA

- контракт соблюден,
- обязательные поля присутствуют,
- типы данных корректны,
- ошибки предсказуемы и информативны,
- логирование/трейсинг позволяет дебажить.
