# 🎯 ДЕНЬ 1: Установка инструментов и основы SQL

## Часть 1: Где и как писать SQL запросы (1 час)

### Что такое SQL?

**SQL (Structured Query Language)** — это язык для работы с базами данных. Как QA-инженер, ты будешь использовать SQL чтобы:
- Проверять корректность данных в базе
- Находить баги в логике приложения
- Готовить тестовые данные
- Проверять результаты работы функций

### Где писать SQL запросы?

У тебя есть 3 варианта (выбери ОДИН для начала):

#### ✅ Вариант 1: SQLite Online (РЕКОМЕНДУЮ ДЛЯ СТАРТА)

**Преимущества:** Не нужно ничего устанавливать, работает сразу в браузере

**Как начать:**
1. Открой браузер
2. Перейди на https://sqliteonline.com/
3. Увидишь интерфейс с тремя областями:
   - Слева: список таблиц
   - Сверху: окно для написания SQL
   - Снизу: результаты выполнения
4. Нажми кнопку "Click to connect" если нужно

**Как выполнить первый запрос:**
```sql
SELECT 'Привет, SQL!' as greeting;
```
1. Напиши это в верхнем окне
2. Нажми кнопку "Run" (или Ctrl+Enter)
3. Увидишь результат внизу

#### Вариант 2: DB Browser for SQLite

**Преимущества:** Профессиональный инструмент, похож на то, что используют в работе

**Установка:**
1. Скачай с https://sqlitebrowser.org/
2. Установи как обычную программу
3. Запусти
4. Создай новую базу данных: File → New Database → укажи имя (например, "test.db")

#### Вариант 3: PostgreSQL (для продвинутых)

Пока **НЕ РЕКОМЕНДУЮ** — слишком сложно для первого дня.

## Часть 2: Основы баз данных (1 час)

### Что такое база данных?

Представь Excel-таблицу. База данных — это набор таких таблиц, которые связаны между собой.

**Пример:**

**Таблица "Пользователи":**
| id | имя | email | возраст |
|----|-----|-------|---------|
| 1 | Иван | ivan@mail.ru | 25 |
| 2 | Мария | maria@mail.ru | 30 |

**Таблица "Заказы":**
| id | user_id | товар | цена |
|----|---------|-------|------|
| 1 | 1 | Книга | 500 |
| 2 | 1 | Ручка | 50 |
| 3 | 2 | Ноутбук | 50000 |

### Основные понятия:

- **Таблица (Table)** — как лист в Excel
- **Столбец (Column/Field)** — вертикальный столбец с однотипными данными
- **Строка (Row/Record)** — одна запись в таблице
- **Первичный ключ (Primary Key)** — уникальный идентификатор записи (обычно id)

## Часть 3: Создание первой таблицы (1 час)

### Синтаксис CREATE TABLE

```sql
CREATE TABLE имя_таблицы (
    название_столбца тип_данных ограничения,
    название_столбца тип_данных ограничения
);
```

### Основные типы данных:

- **INTEGER** — целые числа (1, 2, 100, -5)
- **TEXT** — текст ('Привет', 'test@mail.ru')
- **REAL** — числа с запятой (19.99, 3.14)
- **DATE** — дата ('2024-01-15')
- **BOOLEAN** — правда/ложь (TRUE, FALSE)

### Ограничения (constraints):

- **PRIMARY KEY** — уникальный идентификатор
- **NOT NULL** — поле обязательно для заполнения
- **UNIQUE** — все значения должны быть уникальными
- **DEFAULT** — значение по умолчанию

### Пример 1: Создаем таблицу пользователей

```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name TEXT NOT NULL,
    email TEXT UNIQUE,
    age INTEGER,
    registration_date DATE DEFAULT CURRENT_DATE
);
```

**Что значит каждая строка:**
- `id INTEGER PRIMARY KEY` — уникальный номер пользователя
- `name TEXT NOT NULL` — имя обязательно
- `email TEXT UNIQUE` — email не может повторяться
- `age INTEGER` — возраст (может быть пустым)
- `registration_date DATE DEFAULT CURRENT_DATE` — дата регистрации (автоматически сегодня)


## Часть 5: Чтение данных — SELECT (1 час)

### Базовый синтаксис SELECT

```sql
SELECT столбцы
FROM таблица;
```

### Примеры:

#### Выбрать ВСЕ столбцы и ВСЕ строки:
```sql
SELECT * FROM users;
```
Результат — все 5 пользователей со всеми полями.

#### Выбрать только НЕКОТОРЫЕ столбцы:
```sql
SELECT name, email FROM users;
```
Результат — только имена и email.

#### Выбрать с условием WHERE:
```sql
SELECT * FROM users
WHERE age > 25;
```
Результат — только пользователи старше 25 лет.

#### Несколько условий:
```sql
SELECT name, age FROM users
WHERE age >= 25 AND age <= 30;
```
Результат — пользователи от 25 до 30 лет.
